# 设计文档：导入影片默认屏蔽磁力链接刮削与展示

## 背景与需求
目前，无论是手动输入番号产生的任务，还是拖入/导入本地影片产生的刮削任务，都会自动抓取并展示磁力链接。
用户希望：
1. **导入的影片文件**默认**不需要**刮削与展示磁力链接。
2. **只有**手动输入番号产生的任务（通常以 `__virtual__:` 为前缀）才允许刮削和展示磁力链接。

---

## 方案对比

### 方案 1 (推荐)：在 `ScrapeWorker` 中进行数据源头拦截
在 `gui/scrape_worker.py` 的 `run` 线程中，在获取到 `detail` 元数据后进行拦截。
如果 `self.file_path` 不以 `__virtual__:` 开头（表明是导入的本地文件），则强行清空 `detail["magnets"] = []`。

*   **优点**：
    *   **实现简单**：只修改 `gui/scrape_worker.py` 一个文件，该文件共 147 行，修改极安全，不需要重构。
    *   **源头阻断**：由于直接修改了发射给主界面的 `detail` 数据，UI 渲染、缓存、落盘等所有下游流程天然都将磁力链接视为空，逻辑完全统一。
*   **缺点**：
    *   JavDB 会在抓取详情 HTML 时带上磁力部分，但由于这是一次性请求，无法在 HTTP 层拆分，所以对网络性能无负面影响。

### 方案 2：在 `gui/controller.py` 渲染层进行拦截
在控制器渲染表格的地方，判断 `file_path` 是否是 `__virtual__:`，若不是则强行将磁力表格清空。

*   **优点**：控制渲染展示层。
*   **缺点**：
    *   `gui/controller.py` 超过 300 行（共 915 行），根据项目规则，必须先提议拆分才能修改，增加改动复杂度与风险。
    *   下游的缓存字典或如果后续有磁力导出功能仍会包含磁力，导致行为不一致。
