# JAVDB API Scraper GUI 软件设计规范

本项目旨在基于现有的 `javdb-api-scraper` 核心抓取库，开发一款运行于 Windows 和 macOS 双平台的桌面 GUI 软件。

## 1. 软件概述与核心功能
本软件专为多媒体管理（如 Emby, Jellyfin, Plex, Kodi）用户设计，用于影片本地整理与信息刮削。
*   **拖拽导入**：支持将单个视频文件或整个视频文件夹拖入软件界面。
*   **番号提取**：基于内置的正则表达式算法，从杂乱的影片文件名中自动识别提取番号。
*   **列表校对**：通过可视化表格展示提取到的番号，允许用户在刮削执行前双击直接修改纠错。
*   **异步刮削**：基于 PySide6 的多线程技术异步并发调用后台刮削 API，不卡死 UI 界面。
*   **元数据落盘**：生成标准的 Emby/Kodi 兼容的 XML `.nfo` 元数据文件。
*   **多媒体资产整理**：自动创建 `[番号] 标题` 文件夹，将影片重命名移入其中，并自动下载大封面图为 `poster.jpg`，将预览图存放在新建的 `extrafanart/` 目录下。
*   **代理与 Cookie 管理**：内置 HTTP/Socks5 代理设置（可测试连接）并支持用户配置 JAVDB Cookie 以解锁受限内容。

## 2. 界面设计 (极简白金暗黑风格)
软件使用 PySide6 开发，界面风格遵循 Premium “白金暗黑”（Platinum Noir）视觉设计：
*   **背景色 (Background)**：`#121212`
*   **卡片与面板背景 (Surface)**：`#1E1E1E`
*   **品牌高亮色 (Accent/Platinum)**：香槟金/白金 `#D4AF37`
*   **主文本色**：`#F5F5F7`
*   **次要文本色**：`#8E8E93`
*   **主要部件与布局**：
    *   **左侧配置面板**：数据源选择（JAVDB/JAVBUS）、代理服务器地址与“测试连接”按钮、Cookie 导入文本框、整理目标保存路径选择。
    *   **中部任务区**：首屏带虚线边框的拖拽占位板；导入后转化为扁平化的无边框 QTableWidget，展示文件名、提取的番号（可双击直接编辑编辑）、刮削进度状态（以森林绿/金黄/暗红等状态药丸 Capsule 表示）。
    *   **右侧预览卡片**：圆角卡片渲染的主封面、金色的番号与大标题、分类药丸、横向滚动的预览缩略图网格。

## 3. 技术架构与多线程设计
*   **开发语言与框架**：Python 3.x + PySide6
*   **架构模式**：MVVM
*   **多线程并发控制**：
    *   主线程只负责 UI 的渲染与响应。
    *   引入 `QThreadPool`，默认并发上限设置为 2~3，避免高并发请求导致 IP 被封禁。
    *   每个刮削整理任务封装为 `ScrapeWorker(QRunnable)`，包含刮削请求、元数据解析、图片下载、文件整理。
    *   `ScrapeWorker` 通过自定义 `QObject` 信号与主界面通信，实时反馈进度。

## 4. 代理与 Cookie 的注入方案
*   **全局代理设置**：
    在 GUI 中配置的代理参数（例如 `http://127.0.0.1:7890`）将在初始化 `JavdbAPI` 时，动态注入到 `curl_cffi` 的 `self.session.proxies` 中：
    ```python
    self.session.proxies = {
        "http": proxy_url,
        "https": proxy_url
    }
    ```
    同时，对于 `lib/javdb_adapter.py` 等文件中使用的标准 `requests.get` 进行封装或同样传入该代理参数，确保图片下载正常。
*   **Cookie 管理**：
    用户在界面输入的 Cookie 将更新在内存中并写入 `cookies.json`。刮削初始化时自动调用 `ensure_login()` 或读取该 Cookie 进行接口鉴权。

## 5. 核心逻辑：番号匹配与文件整理

### A. 番号正则捕获算法
1.  **预清洗**：去除常见无用前缀与字幕尾缀，如 `[1080p]`, `[中文字幕]`, `-ch`, `_C`。下划线与空格统一转化为 `-`。
2.  **正则表达式捕获**：
    *   捕获 `(字母)-(数字)`：`r'\b([A-Za-z]{2,5})[-_]?(\d{3,5})\b'`
    *   捕获 FC2 视频：`r'\b(fc2[-_]?ppv[-_]?\d{5,7})\b'`
    *   其他特殊标志匹配。
3.  如果匹配到无连字符的情况（如 `SSIS123`），算法自动转为标准格式 `SSIS-123`。
4.  匹配失败则标记 `[ 待补充 ]`。

### B. Kodi/Emby 兼容性文件整理流程
1.  **路径创建**：清洗非法字符后，创建 `[番号] 标题` 文件夹。
2.  **文件重命名与归类**：将视频文件移动至该文件夹下，重命名为 `番号.后缀`。
    *   *多CD支持*：如果识别出为同一个番号的多 CD 视频（带有 `CD1/CD2` 或 `A/B` 字样），移动到同一文件夹，并重命名为 `番号-CD1.后缀` / `番号-CD2.后缀`。
3.  **生成元数据 NFO**：在视频同级目录下写入 `[番号].nfo`。内容为符合 Kodi / Emby 规范的 XML。
4.  **海报与剧照下载**：
    *   主封面保存为 `poster.jpg`。
    *   横版大图保存为 `fanart.jpg`。
    *   所有的预览图（样品图）下载后保存到新创建的 `extrafanart/` 目录下，依次命名为 `fanart1.jpg`, `fanart2.jpg` 等。

## 6. 异常与边界处理
*   **刮削失败/未找到**：标记状态为“未找到”，不中断其他影片。右侧详情页提供快速纠错。
*   **网络连接超时**：自动使用指数退避算法进行 3 次重试，第 3 次失败后抛出异常，状态置为“网络超时”，保留重试按钮。
*   **磁盘写入冲突**：
    *   目标路径已存在同名影片，若大小一致，则跳过移动直接刮削，避免重复复制。
    *   若大小不一致，则重命名目标文件夹为 `[番号] 标题_1` 进行隔离存放，防止覆盖用户原有珍贵视频。
